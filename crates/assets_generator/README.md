# assets_generator

Automatically generates a Rust enum containing all your game assets based on the files in your assets directory.

## Usage

Create a `build.rs` file in your project root:

```rust
use assets_generator::build_assets_enum;
use std::path::Path;

fn main() {
    build_assets_enum(&Path::new("src").join("assets.rs"), None, None);
}
```

## Parameters

```rust
build_assets_enum(
    dest_path: &Path,      // Output path for the generated file (e.g., src/assets.rs)
    src_path: Option<&Path>, // Optional custom assets directory (defaults to "assets")
    name: Option<String>   // Optional custom enum name (defaults to "FileAssets")
);
```

## Example with Custom Options

```rust
fn main() {
    // Generate a custom enum named "MyAssets" from a custom directory
    build_assets_enum(
        &Path::new("src").join("my_assets.rs"),
        Some(Path::new("my_assets")),
        Some("MyAssets".to_string())
    );
}
```

## Generated Output

Given an `assets/` directory structure:
```
assets/
├── player.png
├── enemies/
│   ├── basic.png
│   └── boss.png
└── audio/
    └── background.mp3
```

The generator creates `src/assets.rs`:

```rust
// This file is autogenerated
use assets_helper::AssetsTrait;

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub enum FileAssets {
    #[allow(dead_code)]
    PlayerPng,
    #[allow(dead_code)]
    EnemiesBasicPng,
    #[allow(dead_code)]
    EnemiesBossPng,
    #[allow(dead_code)]
    AudioBackgroundMp3,
}

impl AssetsTrait for FileAssets {
    fn path(&self) -> &'static str {
        match self {
            FileAssets::PlayerPng => "player.png",
            FileAssets::EnemiesBasicPng => "enemies/basic.png",
            FileAssets::EnemiesBossPng => "enemies/boss.png",
            FileAssets::AudioBackgroundMp3 => "audio/background.mp3",
        }
    }
}
```

## Using the Generated Enum

```rust
use bevy::prelude::*;
use crate::assets::FileAssets;

fn load_assets(asset_server: Res<AssetServer>) {
    // Using the helper method
    let player_handle: Handle<Image> = FileAssets::PlayerPng.load(&asset_server);
    
    // Or using the trait directly
    let audio_handle: Handle<AudioSource> = FileAssets::AudioBackgroundMp3.load(&asset_server);
    
    // Access the path
    let path = FileAssets::PlayerPng.path(); // "player.png"
    
    // Get a scene reference
    let scene_path = FileAssets::Level1Scn.scene(0); // "levels/level1.scn#Scene0"
}
```

## How It Works

1. The build script scans your `assets/` directory recursively
2. Each file is converted to a PascalCase enum variant (e.g., `enemies/basic.png` → `EnemiesBasicPng`)
3. The generated enum implements `AssetsTrait` from the `assets_helper` crate
4. Cargo automatically reruns when files in `assets/` change
